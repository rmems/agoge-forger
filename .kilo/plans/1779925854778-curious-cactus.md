# Plan: Manual GitHub Actions Workflow for Serverless Inference Smoke Tests

## Issue Reference
- GitHub Issue: rmems/agoge-forger#2
- Goal: Add a manual GitHub Actions workflow for short serverless inference checks

## Step 1: Create `.github/workflows/` directory structure
- Create `.github/workflows/serverless_smoke_test.yml`
- Workflow triggers manually only (no push/PR triggers)
- Define inputs: model, workload, max_requests, concurrency, stream, dry_run, upload_artifacts

## Step 2: Create/modify experiment config
- Reuse existing `configs/smoke_test.yaml` or create `configs/serverless_smoke_test.yaml`
- Minimal config for serverless inference smoke test

## Step 3: Create Python runner script
- Create `scripts/serverless_smoke_test.py` as the entry point
- Script will:
  - Accept CLI args matching workflow inputs
  - Save provider info to `provider.json`
  - Save config to `manifest.json`
  - Run smoke check (using existing `smoke_eval.py` logic or new inference check)
  - Save usage snapshots (before/after) and compute delta
  - Output `results.jsonl` and `summary.md`

## Step 4: Create documentation
- Create `docs/serverless_smoke_test.md`
- Explain setup and launch steps from Actions tab
- Document required secrets/setup (out of scope but note what's needed)

## Step 5: Implement workflow steps
- Checkout repository
- Set up Python environment
- Run config check
- Run smoke check
- Upload all expected artifacts

## Artifacts Produced by Workflow
- `manifest.json` - experiment config snapshot
- `provider.json` - runtime environment info
- `usage_before.json` - token usage before
- `usage_after.json` - token usage after
- `usage_delta.json` - usage difference
- `results.jsonl` - per-request results
- `summary.md` - human-readable summary

## Dependencies
- Existing `smoke_eval.py` logic for smoke checks
- Existing `scripts/check_env.py` for environment info
- Pydantic models in `src/agoge_forger/config.py`
